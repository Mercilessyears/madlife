# vue中自定义SVG图标

# 缘由：文件项目中需要的多种类型图标

UI一直用的是element-ui和iview，但是还是发现有少一些自己想要的图标，然后在Iconfont中有很多图标，但是每次单独引入比较麻烦，所以自己写个组件

# 使用

  • # 1.先去Iconfont图库中选图标

    如图： 点击下载素材，选择svg

  • # 2.引入项目

    将下载好的svg放入项目中assets中

  • # 3.编写组件

    新建SvgIcon组件

      <template>
      <svg :class="svgClass" aria-hidden="true" v-on="$listeners">
          <use :xlink:href="iconName"></use>
      </svg>
  </template>

  <script>
  export default {
      name: 'SvgIcon',
      props: {
          iconClass: {
          type: String,
          required: true,
          default: ''
          },
          className: {
          type: String,
          default: ''
          }
      },
      computed: {
          iconName () {
          return `#icon-${this.iconClass}`
          },
          svgClass () {
          if (this.className) {
              return 'svg-icon ' + this.className
          } else {
              return 'svg-icon'
          }
          }
      }
  }
  </script>

  <style scoped>
  .svg-icon {
      width: 1em;
      height: 1em;
      vertical-align: -0.3em;
      fill: currentColor;
      overflow: hidden;
  }
  </style>

    还是在Nuxt.js中,plugins文件夹下

    // svgIcon.js
  import Vue from 'vue'
  import SvgIcon from '../components/public/SvgIcon'

  Vue.component('svg-icon', SvgIcon)

  const req = require.context('@/assets/my-icons/svg', false, /\.svg$/)
  const requireAll = requireContext => requireContext.keys().map(requireContext)
  requireAll(req)

  // nuxt.config.js中添加loader
  module.exports = {
      ...
      plugins: {
          '@/plugins/svgIcon.js'
      },
      ...
      extend (config, ctx) {
          // 排除 nuxt 原配置的影响,Nuxt 默认有vue-loader,会处理svg,img等
          // 找到匹配.svg的规则,然后将存放svg文件的目录排除
          const svgRule = config.module.rules.find(rule => rule.test.test('.svg'))
          svgRule.exclude = [resolve(__dirname, 'assets/my-icons/svg')]
          // 添加loader规则
          config.module.rules.push({
              test: /\.svg$/,
              include: [resolve(__dirname, 'assets/my-icons/svg')],
              loader: 'svg-sprite-loader',
              options: {
              symbolId: 'icon-[name]'
              }
          })
      }
  }

  • # 4.组件使用

因为已经在全局引入该组件了，所以直接引用

<svg-icon :icon-class="doc">

TIP

此处的doc就是assets文件夹中的doc.svg
