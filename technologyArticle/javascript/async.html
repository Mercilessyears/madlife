# ES6 -- “孪生兄弟” async/await

TIP

学海无涯苦作舟！ 守得云开见月明！

# 什么是async

async 是“异步”的简写,async 函数返回的是一个 Promise 对象。async 函数（包含函数语句、函数表达式、Lambda表达式）会返回一个 Promise 对象，如果在函数中 return 一个值，async 会把这个直接量通过 Promise.resolve() 封装成 Promise 对象

async function myAsync () {
    return 'async'
}
const res = myAsync()
console.log(res) //Promise {<resolved>: "async"}

# 什么是await

await 是个运算符，用于组成表达式，await 表达式的运算结果取决于它等的东西。如果它等到的是一个 Promise 对象，等 Promise 对象 resolve，然后得到 resolve 的值，作为 await 表达式的运算结果。

# 两者合体后主要干什么

主要是将 异步 变为 同步，比如一个函数A，由B、C两个函数组成，我要等B完成有返回值后才能开始执行C

async function A(){
   const b = await B()
   console.log(C(b))
}
function B(){
    return new Promise((resolve, reject) => {
        settimeout(() => {
            console.log('我是B内部的值')
            resolve('我是B的返回值')
        }, 2000)
    })
}
function C(c){
    if(!c) {return}
    console.log(c)
}

细心的同学可能发现了，上面这样写还是有一些问题的，什么问题呢？ 问题就在这儿,没有异常捕获

const b = await B() // 我是问题~~
...
function B(){
    return new Promise((resolve, reject) => {
        settimeout(() => {
            console.log('我是B内部的值')
            reject(false)
        }, 2000)
    })
}

WARNING

这样写并不能捕获异常，有人说用try catch就可以了，确实可以，还要另外一种写法就是用链式调用的catch

// ① 
try{
    const b = await B()
}catch(e) {
    console.log(e)
}

// ②
const b = await B().catch((err) => {console.log(err)})
